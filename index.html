O-checker

Veröffentlicht am 2019-04-08 | Bearbeitet am 2019-04-09

O-checker: Detection of Malicious Documents through Deviation from File Format Specifications

概述：

背景：
在目标邮件攻击中，攻击者常常会诱导收信人打开恶意邮件附件，从而造成机密信息泄露等一系列危害。恶意邮件附件中，大部分都是文档文件。和可执行文件不同的是，不能通过检查文件扩展名来判断风险。
主要内容：
o-checker：一种恶意文档文件检测工具。
主要思想：
检查文档文件和文件规范的偏差。
优点：

  1. 速度快。
  2. 稳定性好。在文档中隐藏恶意代码的技术种类相对于恶意代码的种类要少得多，并且文档文件规范比文档处理器也要稳定得多。
  3. 准确率高。在实验中，o-checker获得了96.1%的准确率。

恶意文档文件的结构和一般执行过程：

  • Exploit code：利用文档处理器漏洞的程序。
  • Shell code：创建可执行文件和诱饵文档文件，并启动可执行文件。
  • Executable file
  • Decoy document file

文档结构

RTF

由控制字、控制符和组组成。

  • {} 定义一个组，组可以嵌套。
  • \ 标识一个RTF控制码
  • {\rtf ...} 一个有效的RTF文件都包含这样一个组，{是RTF代码的第一个字符，}表示文件结束符。

CFB

一个包含存储和流的嵌套层次结构的单个文件。与文件结构类比，存储对应目录，流对应文件。
物理结构：一个512字节的头部和一些等长的扇区。用一个32位非负整数标识扇区，成为扇区号。一个流被划分存储到多个扇区中。
头部中定义了FAT和DE的位置。FAT中定义了扇区链。DE中定义管理名字，大小、流，存储的父子关系。

  1. 编码问题

    格式支持：pcm（不压缩）、wav（不压缩，pcm编码）、amr（压缩格式）。推荐pcm 采样率 ：16000 固定值。 编码：16bit 位深的单声道。百度服务端会将非pcm格式，转为pcm格式，因此使用wav、amr会有额外的转换耗时。
