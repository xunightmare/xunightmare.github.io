注意事项：

  1. 编码问题

    格式支持：pcm（不压缩）、wav（不压缩，pcm编码）、amr（压缩格式）。推荐pcm 采样率 ：16000 固定值。 编码：16bit 位深的单声道。百度服务端会将非pcm格式，转为pcm格式，因此使用wav、amr会有额外的转换耗时。

如果音频不是指定的格式，用在线音频格式转换或者格式工厂可能会出现编码错误，进而导致百度语音识别出一堆乱七八糟的东西，最好使用ffmpeg转换：

  1. 搜索模型问题

    搜索模型： 效果同手机百度搜索的语音输入。适合于短语识别，没有逗号。
    输入法模型：效果同百度输入法的语音输入。适合于长句识别，有逗号。

这道题的关键点在于q大约等于10*p，我们可以设q=10*p+r，就有 p*(10*p+r)=N，化简得到10*p^2 + r*p - N = 0，将r看作已知量，这是一个关于p的一元二次方程，解应该为(sqrt(r^2+40N)-r)/20(另外一个解为负数，不需要考虑)，因为这个解一定是能够解出来的，所以我们可以通过爆破的方式来找到r，只要依次满足下面两个条件，那么应该就是我们需要的r：

  1. 开方为整数
  2. 最后解出的p为整数

Misc1

在所有的icmp包中，id为0x930a的包最多，而且长度都为60，数据域的长度都为一个字节，很可疑。先进行过滤，然后发现前面4个包的数据域为0x50, 0x4b, 0x03, 0x04，和zip压缩包的文件标识符一致，在这里应该藏了一个zip压缩包。

继续向后观察，能够找到flag.txt（几个连续的包拼起来），更加确定flag就是藏在压缩包里

用tshark提取出数据：tshark -r pcapng.pcapng -Y "icmp and icmp.ident==0x930a" -T fields -e data | xxd -r -p > flag.zip && xxd flag.zip：

在压缩包中有一个flag.txt文件，但是打开这个文件需要密码，直接用工具爆破就可以了，解出来密码只有一个字符'q'，输入就可以看到flag：
flag{d27ca49cb6f5583ae9f3bb2ec38190de}
