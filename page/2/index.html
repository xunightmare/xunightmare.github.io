mv命令

Posté le 2019-07-07 | Edited on 2019-07-08 | In Linux

mv命令用于为移动文件/目录和重命名文件/目录

定义：

将路径分为path和filename（包含目录）两部分，包含path和filename的写法称为完整路径，而省略了filename的写法称为不完整路径，即：
完整路径：path/filename
不完整路径：path

在介绍这个命令的两种用法之前，首先强调三点理解：

  1. 目录本质上是特殊的文件，所以大部分时候处理方法和文件一致，无需特别区分。
  2. 移动文件/目录和重命名文件/目录本质上都是改变文件的路径，无需特别区分。
  3. 处理优先级：首先尽量认为target是完整路径，先尽量无冲突修改路径，然后考虑覆盖；上面两种情况都不可行的情况下认为target是不完整路径，构造新的完整路径，同样先尽量无冲突修改路径，然后考虑覆盖；如果上面的情况都不行，只能修改失败退出。

基本用法：

工作原理：

  1. 检查源路径是否存在，不存在报错并退出；
  2. 首先认为目标路径是完整路径，首先检查path部分，如果该完整路径的path部分不存在，报错并退出；
  3. 然后检查filename部分，如果不存在，直接修改路径，命令执行成功，如果存在，继续判断；
    1. 完整路径path/filename是文件，如果源路径是文件，则覆盖，如果是文件夹，则报错并退出。
    2. 完整路径path/filename是目录，认为目标路径是不完整路径，即将原来的路径看作新的path，将源文件/目录名看作是filename，重新拼接成完整路径path/filename。新的完整路径如果不存在冲突，直接修改路径，命令执行成功；如果存在冲突，尝试覆盖，目录不能覆盖文件，文件不能覆盖目录，目录不能覆盖非空目录，文件可以覆盖文件。

进阶用法：

  1. 批量移动文件
    1. 正则表达式匹配
  1. 选项
  2. 是否可能导致文件丢失？
  1. 如何区分移动文件和重命名
  2. 如何区分移动文件和目录
  3. 如何区分是否递归移动目录 可能默认就是递归的，不是递归的情况感觉怪怪的
  4. 如果移动的文件正在读写，会出现什么情况。
  5. 如果命令出错后果是什么，会导致文件丢失吗？ 文件名重复、路径不存在、mv命令是如何实现的呢
  6. 想象自己就是计算机，需要判断上面的这些问题，我需要哪些信息呢？

如何导出微信语音

Posté le 2019-07-06 | In 工具

微信语音不支持导出和转发功能，初衷是为了保护用户隐私，预防恶意利用加工语音导致的问题，但是有的时候确实会给用户带来麻烦，这里介绍导出微信语音的思路。

方法一

用另外的设备对语音进行录音，这是最直接的方法。这样获得的语音虽然不是源文件，但是也能够达到目的。

方法二

第二种方法使用微信PC客户端，能够获得语音源文件。

  1. 将语音添加到收藏
  2. “微信客户端 -> 设置 -> 通用设置 -> 微信文件的默认保存位置 -> 打开文件夹 -> 寻找语音文件，寻找语音文件的方法：
    1. 找到保存被收藏的语音文件的目录，我的电脑上（微信2.6.8.52）是：微信文件的默认保存位置\微信用户名\FileStorage\Fav\Temp\字母和数字组成的文件夹名\res
    2. 微信保存的语音文件后缀名为.silk，所以可以在微信文件的默认保存位置搜索后缀名为.silk的文件。可以使用Windows的文件资源管理器搜索*.silk，也可以使用Cmd命令for /r %i in (*.silk) do @echo %i搜索。
  3. 用楼月微信语音播放器（百度就可以下载）可以打开.silk文件并将其导出为.wav文件。

使用scp在Windows和Linux之间传输文件/目录

Posté le 2019-07-06 | In Linux

有时候需要在Windows和Linux系统之间传递文件/目录，如果只是偶尔使用这个功能，专门下载软件太麻烦了，这时候可以用scp命令实现我们的需求。

scp是secure copy的缩写，是Linux系统下基于ssh登录进行的安全文件/目录拷贝命令。

有几个注意事项：

  1. 在传输目录的时候，一定要加上参数-r 否则无法传输成功。
  2. 客户端可以省略username，hostname，previous working directory，只需要填写相对路径就可以了。
  3. 当Windows作为ssh server的时候，有几个注意事项：
    1. Windows10 默认并没有安装ssh服务，通过 “设置 -> 应用 -> 应用和功能 -> 可选功能 -> 添加功能 -> openssh服务端” 来安装ssh服务，然后通过命令net start sshd启动服务，通过命令net stop sshd停止服务。
    2. Windows账号的username可以通过在Cmd或者PowerShell中使用whoami命令查看，密码为Microsoft账号的密码。
    3. Windows的路径写法/c:/path

具体来说，可以分为四种情况:

  1. Linux为ssh client，Windows为ssh server：
    1. 从Linux上拷贝文件到Windows上
    2. 从Windows上拷贝文件到Linux上
  2. Windows为ssh client，Linux为ssh server：
    1. 从Linux上拷贝文件到Windows上
    2. 从Windows上拷贝文件到Linux上

Windows Terminal安装及体验过程记录

Posté le 2019-07-06 | Edited on 2019-07-08 | In 工具

Microsoft在Build 2019开发者大会上发布了Windows Terminal，这是一款新的command line terminal工具。

特点：

  1. 更好看的UI。
  2. 支持多标签。
  3. 支持自定义配色方案。
  4. 能够访问PowerShell、Cmd、WSL等环境。

安装过程：

  1. 提示windows 10版本不够，下载windows10更新助手更新到最新版然后安装
  2. 更新过程中提示VMware Workstation Pro 15.01不兼容，升级到15.10就可以了
  3. 从Microsoft Store中安装Windows Terminal (Preview)，遇到了Microsoft Store无法连接网络的问题，通过 设置 -> 网络和Internet -> 代理 -> 将【使用设置脚本】关闭 解决。

体验：
和Cmd和PowerShell相比，用起来确实舒服很多，但是Preview版本bug还是很多，经常莫名其妙崩溃，非常影响使用体验。
