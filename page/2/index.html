从一个有趣的例子引出C语言缓冲区

从一个有趣的例子引出C语言缓冲区

#include <stdio.h>

int main(){
    printf("I'm ok");
    sleep(5);
    printf("I'm ok2");
    sleep(5);
}

上面的程序，在执行的时候，并不是我刚开始所预期的先输出第一句话，然后休眠5s，然后再输出第二句，然后再休眠5s，而是休眠10s之后再依次输出这两句话。

感觉很奇怪，程序是不是按顺序执行的吗？

去google了一下，加上自己的尝试，终于搞明白了问题所在：标准输出有缓冲区，printf的内容并不会马上打印出来，而是会先暂时保存在缓冲区里。

为什么要这样做呢？
为了提高磁盘的读写效率，提高计算机的运行速度。

如果想要让内容马上打印出来，有什么方法吗？

  1. 用输出的内容将缓冲区填满，这个方法我只是觉得理论上可行，但是并没有实际尝试过，我也不知道缓冲区究竟有多大。
  2. 在想要立即输出的内容后面输出\n，或者调用函数fflush(stdout)
  3. 程序退出的时候会清空缓冲区

关于C语言缓冲区的详细解释，请参考这篇博客：c语言 - 缓冲区详解

为什么我要搭建这个博客+搭建这个博客的过程

其实想要搭建这样一个博客很久了，觉得作为一个程序员，有一个自己域名的博客是一件非常酷的事情，但是因为各种各样的原因，一直拖着没有动手。10月2日是我的生日，那天突然想要发一些分享，但是又不想在QQ空间或者朋友圈发，所以就想要买一个域名，祝自己生日快乐！

下面简单记录我搭建这个博客的过程

0x00 搭建步骤：

  • 创建自己的github pages
  • 在本地用hexo搭建博客
  • 将本地的hexo博客发布到自己的github pages上
  • 用github学生优惠包在namecheap上获得.me的免费域名
  • 设置DNS解析
  • 设置github pages的CNAME文件

0x01 创建自己的github pages

我的理解，github pages其实就是一个特殊的repository，就像create一个普通的repository一样来创建它，唯一不同的在于：这个repository的名字是固定的（user_name.github.io）

创建这个了这个repository之后，就可以通过域名user_name.github.io来访问自己的github pages，其中，刚刚创建的名为user_name.github.io的repository为主页，而其他的页面则是子页面

0x02 在本地用hexo搭建博客

分享一篇教程：知乎专栏：GitHub+Hexo 搭建个人网站详细教程

首先需要进行配置环境（Node.js + git）：

  1. Hexo基于Node.js，安装过程：从官方网站下载对应的版本并安装，安装完成后可以通过Node.js command prompt打开命令行界面，输入命令node -v来判断是否安装成功，因为Node.js还包含了环境变量和npm的安装，所以还需要用命令npm -v来判断npm是否安装成功。
  2. 安装git的过程：到官网下载安装，配置用户名和密码，然后生成密钥文件，在github上配置ssh公钥，最后可以在git bash中输入命令ssh git@github.com来检测是否配置完成。

完成了上面两步，安装Hexo的环境就已经全部搭建完成了。
下面开始安装Hexo：

npm install -g hexo-cli

安装完成后，生成blog文件：

hexo init blog

可以在本地先检测网站的雏形，输入命令：

hexo g 
hexo s

第一句命令的意思是通过source文件夹中的文件生成网页，第二句是启动服务器，我们就可以在浏览器地址栏输入localhost:4000来访问我们的博客。

0x06 设置github pages的CNAME文件

先说设置的方法，我知道的有两种：

  1. 手动在user_name.github.io这个repository中创建一个名为CNAME的新文件，里面第一行写入你的域名
  2. 在user_name.github.io的settings中设置

我对CNAME这个文件的理解，它的作用有两个：1. 允许我们的域名绑定到github pages上；2. 将user_name.github.io这个域名重定向到我们的域名上

但是如果只是手动修改的话，每次推送博客之后，CNAME文件就会被覆盖，解决办法是将CNAME文件写入/blog/source/中，这样每次推送也会保留了（README.md文件也是同理）

完成了上面的步骤，就可以通过自己域名访问自己的博客了，虽然很粗糙，但是成就感满满呢！

0x07 注意事项

  1. 在设置DNS解析的时候，总是存在一定的延迟，我又没有找到判断刚刚的修改是否生效的方法，所以出现了各种奇奇怪怪的问题，走了很多弯路
  2. 浏览器的缓存可能会影响我们的修改，如果出现了无法理解的现象，可以尝试换个设备，换个浏览器试试，还有一个后来才想到的方法，也许可以利用浏览器的无痕模式
