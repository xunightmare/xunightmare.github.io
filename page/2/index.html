C语言实现栈

为什么要使用栈？

栈的特点是先进后出，可以用于保存之前的状态，比如调用子函数，还有迷宫问题中保存路径。

栈需要用的数据结构？

定义栈

定义结点

栈需要实现的一些操作

初始化栈（申请底结点，并初始化数据和top、bottom指针）

出栈（检查栈是否为空，如果不为空，让top指针指向下一个结点，然后释放之前的头结点的内存）

入栈（申请一个结点，设置好data和指针，然后将结点连到栈链表上）

遍历栈（从头结点开始）

遍历栈（从底结点开始，用递归的方法，如果没有到底结点，不断调用自身，如果到了底结点，就返回）

从一个有趣的例子引出C语言缓冲区

从一个有趣的例子引出C语言缓冲区

#include <stdio.h>

int main(){
    printf("I'm ok");
    sleep(5);
    printf("I'm ok2");
    sleep(5);
}

上面的程序，在执行的时候，并不是我刚开始所预期的先输出第一句话，然后休眠5s，然后再输出第二句，然后再休眠5s，而是休眠10s之后再依次输出这两句话。

感觉很奇怪，程序是不是按顺序执行的吗？

去google了一下，加上自己的尝试，终于搞明白了问题所在：标准输出有缓冲区，printf的内容并不会马上打印出来，而是会先暂时保存在缓冲区里。

为什么要这样做呢？
为了提高磁盘的读写效率，提高计算机的运行速度。

如果想要让内容马上打印出来，有什么方法吗？

  1. 用输出的内容将缓冲区填满，这个方法我只是觉得理论上可行，但是并没有实际尝试过，我也不知道缓冲区究竟有多大。
  2. 在想要立即输出的内容后面输出\n，或者调用函数fflush(stdout)
  3. 程序退出的时候会清空缓冲区

关于C语言缓冲区的详细解释，请参考这篇博客：c语言 - 缓冲区详解
