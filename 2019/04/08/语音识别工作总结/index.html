语音识别工作总结

Veröffentlicht am 2019-04-08

0x00 语音分段

输入的音频是一段话，将音频截成多个小段，每一个小段都包含一完整的话。
使用代码：https://github.com/wblgers/py_speech_seg.git

  1. 安装Anaconda：https://www.jianshu.com/p/895bcd4430c9 （注意！最上面的是最老版本的安装文件，我们需要的是下面的新版本）
  2. 执行python multi_detect.py，遇到报错缺少库就使用命令conda install package_name安装
  3. librosa比较特殊，需要用命令conda install -c conda-forge librosa安装
  4. 安装成功之后，在multi_detect.py中填入需要分段的音频路径，执行python multi_detect.py，就会在文件夹save_audio生成分段后的音频。

0x01 使用百度语音识别Python SDK进行语音识别

百度语音识别官方文档：http://ai.baidu.com/docs#/ASR-API/top

注意事项：

  1. 编码问题

    格式支持：pcm（不压缩）、wav（不压缩，pcm编码）、amr（压缩格式）。推荐pcm 采样率 ：16000 固定值。 编码：16bit 位深的单声道。百度服务端会将非pcm格式，转为pcm格式，因此使用wav、amr会有额外的转换耗时。

如果音频不是指定的格式，用在线音频格式转换或者格式工厂可能会出现编码错误，进而导致百度语音识别出一堆乱七八糟的东西，最好使用ffmpeg转换：

  1. 搜索模型问题

    搜索模型： 效果同手机百度搜索的语音输入。适合于短语识别，没有逗号。
    输入法模型：效果同百度输入法的语音输入。适合于长句识别，有逗号。
