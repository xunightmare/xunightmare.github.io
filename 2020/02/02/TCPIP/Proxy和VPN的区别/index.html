首先分别简单介绍Proxy和VPN。

Proxy

客户端不直接访问服务器，而是先将请求发送给一个中间实体——代理服务器，代理服务器代替客户端向服务器发起请求，然后将结果返回给客户端。

代理分为正向代理和反向代理两种：

正向代理

为客户端提供服务，对服务端透明。

作用：

  1. 访问无法访问的网络
  2. 记录上网记录
  3. 匿名
  4. 访问控制

反向代理

为服务器提供服务，对客户端透明。因为这种代理方式和先出现的代理方式正好相反，因此称之为反向代理。

作用：

  1. 负载均衡
  2. 节省公网IP
  3. 保护内网IP
  4. 缓存以加快访问速度

VPN

本质上是一种安全通信的协议，分为两种：

  1. 和登录节点建立隧道，从而访问秘密资源
  2. 和另一个局域网建立隧道

总结

从上面的介绍可以看出：

  1. Proxy和VPN被设计出来的目的是不同的：正向代理主要用于匿名，VPN主要用于建立安全网络连接。
  2. 当VPN用于和登录节点建立隧道时，和正向代理的作用其实是类似的，只是因为需求不同侧重点会有所不同。
